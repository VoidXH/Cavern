# CoverageParser

A .NET 8.0 tool that parses HTML code coverage reports (generated by [reportgenerator](https://github.com/danielpalme/ReportGenerator) from Cobertura XML data) and converts them into structured JSON for programmatic analysis.

## ⚠️ Disclaimer

> **This project is entirely AI-generated using [Poor Man's AI](https://github.com/VoidXH/Poor-Man-s-AI).**

## Overview

CoverageParser reads HTML coverage reports produced by the standard .NET coverage pipeline and extracts:

- **Summary statistics** — total assemblies, classes, files, coverage date
- **Line coverage** — covered, uncovered, coverable, and total line counts with percentages
- **Branch coverage** — covered branches, total branches, and coverage percentages
- **Assembly-level breakdown** — per-assembly line and branch coverage
- **Class-level breakdown** — per-class line and branch coverage with file paths
- **Method-level details** — per-method line coverage, branch coverage, C.R.A.P. score, and cyclomatic complexity
- **Risk hotspots** — methods flagged with high C.R.A.P. (Change, Risk, Acceptable Points) scores and complexity

## Input / Output

| **Input** | A directory containing an HTML coverage report — specifically `index.html` and one or more `Cavern_*.html` class detail files (produced by `reportgenerator`) |
| **Output** | A JSON file containing structured coverage data at summary, assembly, class, and method levels |

## Usage

### Command Line

```
CoverageParser.exe [reportDirectory] [outputFile]
```

| Argument | Default | Description |
|---|---|---|
| `reportDirectory` | `TestResults/HtmlReport` | Path to the directory containing the HTML coverage report |
| `outputFile` | `coverage.json` | Path for the JSON output file |

### Example

```bash
CoverageParser.exe TestResults/HtmlReport coverage.json
```

### Full Pipeline

CoverageParser is intended to be used as part of the test-and-report pipeline:

```batch
:: 1. Run tests with coverage collection
dotnet test /p:CollectCoverage=true ^
  /p:CoverletOutputFormat=cobertura ^
  /p:CoverletOutput=TestResults\coverage

:: 2. Generate HTML report from Cobertura XML
reportgenerator -reports:TestResults\coverage.cobertura.xml ^
  -targetdir:TestResults\HtmlReport

:: 3. Parse the HTML report into JSON
CoverageParser.exe TestResults\HtmlReport coverage.json
```

### Field Descriptions

#### `summary`

| Field | Type | Description |
|---|---|---|
| `parser` | string | Name of the coverage parser that produced the source report (e.g., "Cobertura") |
| `assemblyCount` | int | Total number of assemblies covered |
| `classCount` | int | Total number of classes covered |
| `fileCount` | int | Total number of source files covered |
| `coverageDate` | string | Timestamp when the coverage data was collected |
| `lineCoverage` | CoverageStats | Aggregate line coverage statistics |
| `branchCoverage` | CoverageStats | Aggregate branch coverage statistics |

#### `CoverageStats`

| Field | Type | Description |
|---|---|---|
| `covered` | int | Number of covered items |
| `uncovered` | int | Number of uncovered items |
| `coverable` | int | Total number of coverable items |
| `total` | int | Total lines/items in the source |
| `percentage` | decimal | Coverage percentage (e.g., `25.6` for 25.6%) |

#### `assembly`

Each entry contains:

| Field | Type | Description |
|---|---|---|
| `name` | string | Assembly name (e.g., "Cavern") |
| `lineCoverage` | CoverageStats | Line coverage for this assembly |
| `branchCoverage` | CoverageStats | Branch coverage for this assembly |
| `classes` | ClassReport[] | Per-class coverage details |

#### `ClassReport`

| Field | Type | Description |
|---|---|---|
| `fullName` | string | Fully qualified class name |
| `assembly` | string | Containing assembly name |
| `filePath` | string | Source file path |
| `lineCoverage` | CoverageStats | Line coverage for this class |
| `branchCoverage` | CoverageStats | Branch coverage for this class |
| `methods` | MethodReport[] | Per-method coverage details |

#### `MethodReport`

| Field | Type | Description |
|---|---|---|
| `name` | string | Method name |
| `line` | int | Line number where the method is defined |
| `branchCoverage` | decimal | Branch coverage percentage |
| `crapScore` | decimal | C.R.A.P. score (higher = riskier, less tested complex code) |
| `cyclomaticComplexity` | int | Cyclomatic complexity |
| `lineCoverage` | decimal | Line coverage percentage |

#### `riskHotspots`

Methods that the coverage report flagged as high-risk, based on their C.R.A.P. score and cyclomatic complexity.

| Field | Type | Description |
|---|---|---|
| `assembly` | string | Containing assembly |
| `class` | string | Containing class |
| `method` | string | Method name |
| `crapScore` | decimal | C.R.A.P. score |
| `cyclomaticComplexity` | int | Cyclomatic complexity |

## Architecture

```
CoverageParser/
├── Program.cs                 # Entry point — orchestrates parsing and JSON output
├── Models/                    # Data models for the parsed output
│   ├── CoverageReport.cs      # Top-level report container
│   ├── CoverageStats.cs       # Reusable coverage statistics
│   ├── ReportSummary.cs       # Overall summary
│   ├── AssemblyReport.cs      # Per-assembly report
│   ├── ClassReport.cs         # Per-class report
│   ├── MethodReport.cs        # Per-method report
│   └── RiskHotspot.cs         # High-risk method entries
├── Parsers/                   # HTML parsing logic
│   ├── ParserUtils.cs         # Shared parsing utilities
│   ├── IndexHtmlParser.cs     # Parses index.html (summary, assemblies, classes, hotspots)
│   └── ClassHtmlParser.cs     # Parses individual class HTML files (methods)
└── CoverageParser.csproj      # Project file (.NET 8.0, HtmlAgilityPack)
```

### How It Works

1. **`IndexHtmlParser`** reads `index.html` to extract the overall summary (assembly/class/file counts, coverage date), line/branch coverage statistics, assembly-level data, class-level data, and the risk hotspots table.

2. **`ClassHtmlParser`** iterates over all `Cavern_*.html` class detail files, extracting per-class line/branch coverage and the method-level metrics table (branch coverage, C.R.A.P. score, cyclomatic complexity, line coverage).

3. The populated `CoverageReport` object is serialized to indented JSON with UTF-8 encoding.
